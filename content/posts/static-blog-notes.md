# 为什么我喜欢静态博客

静态博客的魅力在于简单。文章是文件，页面是代码，发布就是把它们放到一个可以访问的地方。

## 少一点基础设施

对于个人项目来说，很多时候并不需要复杂的服务端能力。只要页面能够展示文章、链接能够正常访问、内容能够稳定保存，就已经足够开始写作。

静态网站还带来了一些实际好处：

1. 部署成本低，GitHub Pages 和 EdgeOne Pages 都可以直接托管；
2. 没有数据库维护和服务器更新的负担；
3. 内容与界面分离，换样式不会影响文章本身。

## 文件就是内容

一篇文章可以只是一个普通的 Markdown 文件：

```markdown
# 一篇新的文章

这里是正文内容。
```

把文章写作变成文件管理之后，发布流程会清晰很多。配置文件负责标题、日期和标签，Markdown 负责真正需要阅读的内容。

这并不意味着静态网站只能很简陋。好的排版、合理的留白和清晰的导航，同样可以让一个小网站拥有完整的阅读体验。
